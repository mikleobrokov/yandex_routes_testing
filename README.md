## 4. Список основных баг-репортов

### 🔴 Критические дефекты

| ID | Название |
|----|---------|
| **БР-12** | Пропадает форма бронирования при удалении адресов "Откуда" и "Куда" |
| **БР-13** | Не отображаются все доступные машины на карте (только выбранного тарифа) |
| **БР-14** | Иконки машин наслаиваются на элементы карты и направлены в одну сторону |
| **БР-15** | При бронировании не увеличивается иконка ближайшей машины |
| **БР-16** | Иконки машин на карте не реагируют на нажатие |
| **БР-19** | В окне забронированной машины отображается название тарифа вместо марки |
| **БР-22** | Не открывается окно отмены поездки при нажатии на кнопку отмены |
| **БР-24** | Не отображаются последние 4 цифры добавленной карты |
| **БР-44** | Не открывается окно добавления прав при нажатии кнопки бронирования |
| **БР-45** | При отсутствии способа оплаты кнопка бронирования сразу бронирует машину |

### 🟠 Высокий приоритет

| ID | Название |
|----|---------|
| **БР-23** | Не проставляются пробелы в номере карты после ввода 12 символов |
| **БР-25** | Поля в окне добавления карты заполнены данными предыдущей карты |
| **БР-26** | Поля не очищаются после отмены добавления карты |
| **БР-27** | Невозможно добавить больше 15 карт |
| **БР-28** | Можно добавить две одинаковые карты |
| **БР-29 – БР-42** | Множественные дефекты валидации полей "Номер карты" и "Код" |

## 5. Общие выводы и рекомендации

### 5.1 Критическая стабильность интерфейса
Необходимо исправить ошибку **БР-12**, связанную с исчезновением формы бронирования. Это делает невозможным завершение заказа при стандартных действиях пользователя.

### 5.2 Функциональность карты
Требуется доработка отображения иконок машин (**БР-13**, **БР-14**) и их интерактивности (**БР-15**, **БР-16**). Навигационная карта является ключевым элементом выбора транспорта.

### 5.3 Логика бронирования
Нарушена бизнес-логика при отсутствии обязательных данных (**БР-44**, **БР-45**). Пользователь может либо "застрять" на этапе, либо, наоборот, оформить заказ без оплаты.

### 5.4 Валидация данных
Поля ввода банковской карты пропускают невалидные символы (**БР-29 – БР-42**), что может привести к ошибкам при оплате и несоответствию требованиям безопасности.
